Saya pada hari ini sudah mempelajari dasar-dasar backend, dan saya akhirnya mulai mengerti apa sebenarnya yang terjadi di balik sebuah aplikasi. Dulu saya hanya melihat tampilan saja dan mengira semuanya terjadi di depan layar, tapi setelah belajar hari pertama ini, saya baru paham bahwa ada bagian penting yang bekerja diam-diam di belakang, yaitu backend. Backend inilah yang mengatur alur data, memproses logika, dan memastikan setiap permintaan dari pengguna bisa dijawab dengan benar.

Saya juga mulai mengerti bahwa backend itu seperti otak yang mengatur semuanya. Kalau frontend adalah tampilan yang dilihat pengguna, maka backend adalah bagian yang memastikan tampilan itu bisa menampilkan data yang benar. Misalnya ketika pengguna menekan tombol “login”, yang terlihat hanyalah form di layar, tapi sebenarnya frontend sedang mengirim data ke backend. Lalu backend mengecek apakah data itu valid, lalu mengirimkan respon kembali ke frontend. Dari sini saya mulai bisa membayangkan bagaimana hubungan backend dan frontend itu saling melengkapi. Tidak bisa hanya mengandalkan salah satu, karena keduanya memang harus bekerja sama.

Hari ini juga saya belajar tentang apa itu server. Ternyata server bukan sekadar komputer di suatu tempat, tapi sebuah layanan yang siap menerima request dari pengguna kapan saja. Saya jadi tahu bahwa setiap kali kita membuka aplikasi, sebenarnya kita sedang “berbicara” dengan server dan serverlah yang menjawab.

Saya juga mempelajari apa itu API. Saya baru sadar kalau API itu semacam jembatan yang menghubungkan antara frontend dan backend. Dengan API, frontend bisa minta data ke backend, dan backend bisa mengirimkan jawaban dalam bentuk yang bisa dipahami oleh frontend.

Selain itu, saya belajar tentang method HTTP seperti GET, POST, PUT, dan DELETE. Ternyata method-method ini punya peran masing-masing. GET untuk mengambil data, POST untuk mengirim atau menyimpan data, PUT untuk memperbarui, dan DELETE untuk menghapus. Saya juga dikenalkan dengan HTTP status code seperti 200, 404, dan 500. Dari sini saya mulai paham bagaimana sebuah server memberi tanda apakah permintaan sukses atau ada masalah.

Walaupun ini baru hari pertama, saya merasa sudah dapat gambaran besar tentang dunia backend. Masih banyak yang belum saya pelajari, tapi fondasinya sudah mulai terbentuk. Saya jadi lebih semangat untuk lanjut ke materi berikutnya.
